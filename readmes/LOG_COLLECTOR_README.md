# X-Ray Log Collector

Расширенная функциональность для сбора логов X-Ray#### Прямой запуск коллектора
```bash
python collect_logs.py
```

#### Доступные команды в launcher:
1. **Стандартный коллектор** - использует системный SSH
2. **Enhanced коллектор** - использует Paramiko для надежности  
3. **Анализ ранее собранных логов** - объединение и анализ без нового сбора
4. **Создание примера SSH конфигурации**

#### Только анализ без сбора:
```bash
python log_merger.py
```

#### Тестирование системы:
```bash
python test_system.py      # Полное тестирование
python test_ssh.py         # Только SSH соединения
```даленных серверов через SSH.

## 🚀 Новые возможности

### Автоматический сбор логов
- Подключение к удаленным серверам по SSH
- Извлечение логов из Docker контейнеров
- Автоматическое скачивание логов на локальную машину
- Поддержка как стандартного SSH, так и Paramiko для надежности

### Поддерживаемые режимы
1. **Стандартный коллектор** - использует системный SSH через subprocess
2. **Enhanced коллектор** - использует библиотеку Paramiko для более надежных соединений

## ⚙️ Настройка

### 1. SSH конфигурация
Создайте файл `~/.ssh/config` (Windows: `C:\Users\username\.ssh\config`) с конфигурацией ваших серверов:

```ssh-config
# VPS Node 1
Host node1
    HostName 1.2.3.4
    User root
    Port 22
    IdentityFile ~/.ssh/id_rsa

# VPS Node 2  
Host node2
    HostName 5.6.7.8
    User ubuntu
    Port 2222
    IdentityFile ~/.ssh/id_ed25519

# OpenWRT Router (будет исключен автоматически)
Host rtr
    HostName 192.168.1.1
    User root
    Port 22
```

**Важно**: Роутеры и локальные IP адреса автоматически исключаются из обработки:
- Хосты с именами `rtr`, `router`, `gateway`
- IP адреса из локальных сетей (192.168.x.x, 10.x.x.x, 172.16-31.x.x)
- Хост с IP `192.168.1.1` (OpenWRT по умолчанию)

### 2. SSH ключи
Убедитесь, что у вас настроены SSH ключи для всех серверов:
```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
ssh-copy-id user@hostname
```

```
Выберите опцию "3. Сбор логов с удаленных серверов"

### Прямой запуск коллектора
```bash
python collect_logs.py
```

### Параметры командной строки

#### Стандартный коллектор
```bash
python log_collector.py [параметры]
```

#### Enhanced коллектор
```bash
python enhanced_log_collector.py [параметры]
```

#### Доступные параметры:
- `--logs-dir ./logs` - локальная директория для сохранения логов
- `--container remnanode` - имя Docker контейнера с логами
- `--exclude host1 host2` - хосты для исключения из обработки
- `--cleanup` - автоматически удалять временные файлы на удаленных серверах
- `--all` - обработать все доступные хосты без интерактивного выбора
- `--create-config` - создать пример SSH конфигурации
- `--no-paramiko` - принудительно использовать стандартный SSH

## 🔧 Процесс работы

1. **Инициализация**
   - Определение ОС клиента
   - Чтение SSH конфигурации
   - Автоматическое исключение роутеров и локальных IP

2. **Выбор хостов**
   - Интерактивный выбор хостов для обработки
   - Возможность выбрать все хосты или отдельные
   - Фильтрация локальных сетей

3. **Сбор логов с каждого хоста**
   - SSH подключение к серверу
   - Проверка существования Docker контейнера
   - Копирование лога из контейнера в `/tmp/`
   - Скачивание лога на локальную машину
   - Опциональная очистка временных файлов

4. **Автоматический анализ**
   - Объединение логов с нескольких серверов
   - Сортировка по времени
   - Запуск графического интерфейса анализа
   - Возможность выбора режима анализа (GUI, сводка, онлайн)

## ⚠️ Устранение неполадок

### SSH соединение не устанавливается
1. Проверьте SSH конфигурацию: `ssh -T hostname`
2. Убедитесь, что SSH ключи настроены правильно
3. Проверьте доступность сервера: `ping hostname`

### Docker контейнер не найден
1. Проверьте имя контейнера: `docker ps -a`
2. Убедитесь, что контейнер запущен
3. Используйте правильное имя контейнера с параметром `--container`

### Paramiko недоступен
1. Установите Paramiko: `pip install paramiko`
2. Или используйте стандартный режим с `--no-paramiko`

### Логи не скачиваются
1. Проверьте права доступа к директории `/tmp/` на удаленном сервере
2. Убедитесь, что на локальной машине есть права записи в `./logs/`
3. Проверьте свободное место на диске
