✅ Цель: С клиента (где уже есть SSH-ключи ко всем VPS) подключиться по SSH к каждой ноде (кроме OpenWRT), вытащить нужные логи и сохранить их локально с читаемыми названиями.
🔧 Шаги выполнения скрипта:
1. Инициализация
Определить ОС клиента (Windows или Linux) — для правильного пути к ~/.ssh/config (например, C:\Users\user\.ssh\config).

Считать ~/.ssh/config, извлекая все Host кроме 192.168.1.1 (OpenWRT).

2. Запрос пользователя
Показать список доступных хостов (Host из ssh config).

Спросить, откуда пуллить логи:

❏ С одной или нескольких нод (по выбору).

❏ Со всех доступных (кроме исключений, напр. OpenWRT).

3. Удалённое выполнение команд
Для каждой выбранной ноды:

Подключиться по SSH.

Выполнить команды:

Проверка, существует ли контейнер (по имени remnanode).

Выполнить копирование логов из Docker-контейнера в /tmp или другую временную папку:

```bash
docker cp remnanode:/var/log/supervisor/xray.out.log /tmp/xray_$HOST.log
```
(Или использовать docker exec для чтения лога напрямую, если копировать нельзя.)

4. Копирование логов на клиент
Скопировать файл /tmp/xray_$HOST.log по scp или rsync на клиент в локальную папку (например, ./logs/).

Назвать файлы понятно: xray_node1.log, xray_node2.log и т.д.

5. Очистка (опционально)
Удалить временные логи на удалённой стороне (rm /tmp/xray_*.log) — по подтверждению.
